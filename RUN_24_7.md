# 🟢 تشغيل البوت ٢٤ ساعة على Raspberry Pi (systemd)

هذه الطريقة تجعل البوت:
- يبدأ **تلقائياً** عند تشغيل الـPi.
- **يعيد تشغيل نفسه** خلال ٥ ثوانٍ لو توقف لأي سبب.
- يستمر شغّالاً حتى لو أُغلق الترمنال أو جلسة SSH.

> ملف الخدمة الجاهز موجود في المستودع: [`tiktok-bot.service`](tiktok-bot.service)

---

## 1) أوقف أي نسخة شغّالة + اسحب آخر تحديث
```bash
pkill -f tiktok_info_bot.py
cd ~/informationtiktok
git pull origin main
```

## 2) أنشئ خدمة systemd
انسخ هذا **كاملاً** والصقه (يبني الملف ويفعّله):
```bash
sudo tee /etc/systemd/system/tiktok-bot.service > /dev/null << 'EOF'
[Unit]
Description=TikTok Info Telegram Bot
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=abdalwahab
WorkingDirectory=/home/abdalwahab/informationtiktok
ExecStart=/home/abdalwahab/informationtiktok/venv/bin/python tiktok_info_bot.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable tiktok-bot
sudo systemctl start tiktok-bot
```

> ⚠️ عدّل `User` والمسارات إن كان اسم المستخدم أو مكان المشروع مختلفاً عندك.

## 3) تأكد أنه يعمل
```bash
sudo systemctl status tiktok-bot
```
المفروض تشوف **`active (running)`** بالأخضر. اضغط `q` للخروج.

---

## 🛠️ أوامر الإدارة
| الأمر | الوظيفة |
|---|---|
| `sudo systemctl restart tiktok-bot` | إعادة تشغيل البوت |
| `sudo systemctl stop tiktok-bot` | إيقافه |
| `sudo systemctl start tiktok-bot` | تشغيله |
| `sudo systemctl status tiktok-bot` | حالته |
| `sudo systemctl disable tiktok-bot` | منعه من البدء تلقائياً |
| `journalctl -u tiktok-bot -f` | مشاهدة السجل المباشر (الأخطاء) |

---

## 🔄 بعد أي تحديث للكود مستقبلاً
```bash
cd ~/informationtiktok && git pull origin main && sudo systemctl restart tiktok-bot
```
